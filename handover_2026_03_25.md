# GlobeTracker Outstanding Handover

Date: 2026-03-25

This handover now tracks only open work.
Completed cleanup and already-implemented architecture were removed.

## Current Operating Reality

- production uses `web.1` + `worker.1`
- the poller is embedded in the worker
- local development needs:
  - Puma
  - Sidekiq
  - AnyCable

## What Is Open

### Frontend

Main gap:

- the app still surfaces important things independently
- the camera/live-stream layer still feels like a plain feed instead of a real-time observation surface

Immediate frontend approach:

1. create one shared selected-context state
2. render a unified right-panel context from it
3. start with news, insights, and chokepoints
4. then expand to other node types

Parallel frontend priority:

1. improve camera ranking and presentation
2. promote truly live/fresh streams above passive webcams
3. make live observation useful as corroboration, not just ambience

### Product Layer

Bookmark:

- add a newsroom `leads` layer above raw insights
- each lead should show `what happened`, `why it matters`, `evidence`, `downstream exposure`, and `unknowns`

### Relationships

Current relation graph has:

- `theater_pressure`
- `flow_dependency`
- `downstream_exposure`
- `operational_activity`
  - ships -> chokepoints / submarine cables
  - flights -> theaters / stressed airports / bases
  - jamming + NOTAM evidence where present

Next global builders:

1. stronger `local_corroboration`
2. actor / operator normalization

### Retrieval / Data Expansion

Now available and not yet wired:

- `MEDIA_CLOUD_TOKEN`
- `RELIEF_WEB_APP_NAME`

Most important missing anchor data:

- ports
- cable landing stations
- ASN / ISP metadata
- satellite observation semantics

### Runtime Cleanup

Still unresolved:

- whether the dedicated `poller` compatibility path should remain
- whether local runtime/docs should be simplified around the real Puma + Sidekiq + AnyCable setup

## Suggested Next Sequence

1. finish the first selected-context right-panel slice
2. expose relation-backed node context
3. strengthen `local_corroboration`
4. wire `MediaCloud`
5. wire `ReliefWeb`
